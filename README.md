# Propane Market Intelligence — RAG + Forecasting Hybrid

**[📄 View the full notebook in your browser](https://sharma-tadepalli.github.io/propane-market-rag/)** — results, charts, and code, no Jupyter required.

A framework-free Retrieval-Augmented Generation (RAG) system for energy market analysis, built from first principles: **no LangChain, no LlamaIndex** — every component (chunking, embedding, retrieval, grounding, evaluation) is explicit and inspectable.

The assistant answers questions about the U.S. propane market using only a curated document corpus, cites its sources inline, and **declines to answer when the information isn't in the corpus** — verified by negative tests. A companion time-series model forecasts price direction; the RAG layer explains the fundamentals behind the move. *The forecast says WHAT, the RAG explains WHY.*

## Architecture

```
                ┌─────────────────────────────┐
                │  Document Corpus            │
                │  EIA-style market notes,    │
                │  seasonal outlooks (.txt)   │
                └──────────┬──────────────────┘
                           │ 1. Chunk (250-word windows, 50-word overlap)
                           ▼
                ┌─────────────────────────────┐
                │  Embeddings                 │
                │  sentence-transformers      │
                │  all-MiniLM-L6-v2 (local)   │
                └──────────┬──────────────────┘
                           │ 2. Index
                           ▼
                ┌─────────────────────────────┐
User question ─▶│  Vector Store (ChromaDB)    │
                │  top-k cosine retrieval     │
                └──────────┬──────────────────┘
                           │ 3. Retrieved chunks + question
                           ▼
                ┌─────────────────────────────┐
                │  LLM (Claude API)           │
                │  grounded answer + inline   │
                │  [doc_id] citations         │
                └─────────────────────────────┘
```

## Key results

| Check | Result |
|---|---|
| Retrieval hit-rate@4 on gold QA set | **4/4 (100%)** |
| Groundedness (LLM-as-judge) | **GROUNDED** on all probes |
| Negative test ("WTI crude price?" — not in corpus) | **Correctly declines**, suggests what document would answer it |
| Forecast MAE vs. naive t-1 baseline | **0.0079 vs. 0.0156 $/gal** (~2× better) |

## Design decisions

| Decision | Choice | Rationale |
|---|---|---|
| Embeddings | `all-MiniLM-L6-v2` (local, free) | Fast, zero API cost, strong baseline; would benchmark against `text-embedding-3-small` / Azure embeddings in production |
| Vector store | ChromaDB (in-memory) | Zero-ops for a prototype; production path → Azure AI Search, pgvector, or Pinecone |
| Chunking | ~250 words, 50-word overlap | Market-report paragraphs are self-contained; overlap prevents boundary loss |
| Generation | Claude (Anthropic API) | Strong grounding behavior; a swap-in cell is provided for Azure OpenAI |
| Hallucination control | Strict system prompt + inline `[doc_id]` citations + explicit "say so if not in context" rule | Verified with negative tests |
| Evaluation | Retrieval hit-rate on gold QA + LLM-as-judge groundedness | Production path → RAGAS-style faithfulness/relevance metrics with SME review |

## Why framework-free?

Frameworks are excellent for shipping fast — but for *learning* and for *debugging enterprise deployments*, you need to know what every layer actually does. This project keeps the full RAG pipeline under ~150 lines of readable code, so every retrieval distance, every prompt token, and every evaluation verdict is visible.

## Quickstart

```bash
pip install -r requirements.txt
jupyter notebook notebooks/propane_market_rag.ipynb
```

You'll need an [Anthropic API key](https://console.anthropic.com/) — the notebook reads it from Colab secrets or prompts for it securely (never hardcoded). The sample corpus is generated by the notebook itself, so it runs end-to-end out of the box. To use real data, drop EIA weekly propane prices into `propane_weekly.csv` and your own market documents into `corpus/`.

## Production roadmap

1. **Ingestion:** scheduled pipeline pulling EIA/NOAA releases; document versioning; PDF parsing for STEO reports
2. **Retrieval quality:** hybrid search (BM25 + dense), cross-encoder reranking, date-metadata filtering so "this winter" retrieves the right season
3. **Evaluation at scale:** RAGAS-style metrics, curated eval set with SME review, regression tests on every corpus or prompt change
4. **Agentic extension:** tool-use so the assistant queries the price database (structured) *and* the corpus (unstructured)
5. **Deployment:** Azure AI Search as vector store, managed identity for keys, monitoring for retrieval drift
